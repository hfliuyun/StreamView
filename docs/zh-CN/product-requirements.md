# StreamView 产品需求

状态：设计访谈结束后已经明确确认的首个里程碑正式基线。英文 [Product Requirements](../product-requirements.md) 是规范性来源，本页是同步维护的中文说明。

## 目标与用户

StreamView 是面向音视频开发者、协议分析人员和高级用户的只读桌面二进制分析器。它从容器、轨道和 sample 一直深入到规范定义的编解码字段，同时保留返回未经修改源文件的精确 byte/bit 映射。

## 首个里程碑输入

- 完整、可随机访问的本地文件。
- H.264 Annex B 裸 elementary stream。
- AAC ADTS 裸 elementary stream。
- 包含 AVC 和 AAC 的普通非分片 ISO BMFF MP4/MOV。

网络 URL、RTP/RTSP、HLS/DASH、采集设备、管道、标准输入和网络抓包后置。

## 正式格式支持

- H.264 Baseline、Main 和 High 的结构语法，重点覆盖常见 8-bit 4:2:0 内容：start code、NAL unit、EBSP/RBSP 映射、SPS、PPS、SEI 和 slice header。
- ADTS 及 MP4 AudioSpecificConfig 中的 AAC-LC 结构语法。
- 递归解析普通非分片 MP4/MOV box、轨道、sample table、`avcC` 和 `esds`，并可从容器 sample 导航到 H.264 或 AAC 语法。

fragmented MP4、MPEG-TS、Matroska/WebM、H.264 CAVLC/CABAC 宏块与残差解析、AAC Huffman 频谱负载解析及其他 profile 后置。首个里程碑不包含播放和完整音视频解码。

## 分析行为

- 每个语法字段显示解码值、类型、位宽、源位置、自编简释和带版本的规范引用。
- 字段可以映射到多个绝对源区间，同时具有所在逻辑视图的坐标。
- 保持映射的转换不会隐藏 H.264 防竞争字节等被排除内容，而是保留并标记。
- 格式探测提供候选规则、依据和置信度；用户始终可以选择其他兼容规则。
- 非法、截断和不支持语法生成带源位置的诊断，同时保留合法部分结果，不静默猜测字段值。
- 媒体源严格只读，不提供编辑和回写。

## 桌面工作区

- 分析树显示容器、轨道、sample、编解码单元和字段。
- 虚拟化原始数据视图提供十六进制、二进制和组合模式，并精确高亮 bit。
- 字段检查器显示值、位宽、绝对与逻辑坐标、含义、规范引用和诊断。
- 字段和对应源区间之间双向联动。
- 原始数据的每个 byte 单元从左到右分成八个相等的 source-bit 命中区域，对应从最高
  有效位 bit 0 到最低有效位 bit 7。选择字段时，所有映射 span 在翻页后仍保持高亮，
  并自动显示第一个 span。
- Selection 绑定当前 source identity。成功替换 session 时清除旧 selection；替换失败
  时保留当前 session、渲染内容和 selection。
- 可以从 MP4 sample 元数据进入其中的编解码语法，并返回容器位置。

## 大型文件

- 至少检查 100 GB 文件，同时内存不能随源文件大小线性增长。
- 在普通本地存储条件下，约两秒内给出可用的初始视图。
- 按需物化有界区域，并在后台渐进建立索引。
- 扫描结果逐批出现；长任务显示进度、允许取消并保留已完成结果。
- 跳转到已知 offset 不等待完整文件扫描。

## 会话与导出

- 保存会话与只读媒体源分离，保存源身份、精确规则版本、书签、注释和导航状态。
- 重新使用已保存位置前，通过源指纹检测媒体是否变化。
- 可重建的大型缓存不写入紧凑会话文件。
- 分层 JSON 导出是期望功能，但不阻塞首个里程碑。

## 规则系统

- 内置规则和安装规则使用同一套沙箱化 C 风格声明语言。
- 规则有界、确定、只读、可取消，不能任意访问宿主、网络、进程、指针或原生库。
- 规则定义保留源映射的逻辑视图、安全惰性边界和渐进索引。
- 应用、语言和规则包独立版本化；保存会话锁定精确规则版本。
- 官方规则随应用发布；首版只从本地文件或目录安装其他规则，没有市场和自动更新。
- 详细语言与规则包契约见[格式定义语言说明](format-language/README.md)。

## 平台、实现与发布

- 一级验证平台：Windows 11 x64、macOS 14+ Apple Silicon、Ubuntu 24.04 LTS x64。
- 二级兼容目标：Windows 10、Intel Mac 和其他现代 x64 Linux 发行版。
- 使用 C++20 解析核心、Qt 6 Widgets 桌面界面和 CMake；Python 只用于开发与测试工具。
- GitHub Actions 对每次改动构建并测试一级平台矩阵，从发布 tag 生成平台产物。
- 首批版本可以发布未签名便携包；Windows 签名和 macOS notarization 后置。
- StreamView 与官方规则采用 MIT；Qt 按适用开源许可证动态链接，除非重新评估分发策略，否则避开仅 GPL 的 Qt 模块。
